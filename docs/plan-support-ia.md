# Plan Support Information Architecture

## Changes (v2)

### Removed from Support drawer
- **Deal Strategy** — overlaps with Objective/POV/Plan cards on the main canvas.
- **Enablement** — low engagement; seller/engineer prep tracked elsewhere.
- **Sizing Inputs** — folded into commercial planning outside Support.

### Moved
- **Talk Tracks** → Sales tab (was in Support > Positioning). Title: "Talk Tracks", subtitle: "Customer outcome narrative". Falls back to 4 deterministic bullets (Reduce / Accelerate / Improve / De-risk) when no business package is available.

### Renamed
- **Delivery Assets** → **Delivery** (subtitle: "Discovery agenda, workshop plan, pilot scope")
- **Open Questions** → **Required info from customer** (subtitle: "Items to validate before advancing")

### Final Support drawer section order
1. Value Hypothesis
2. KPIs
3. Delivery
4. Required info from customer

### Behavior
- First section expanded by default; all others collapsed.
- Support trigger button ("Support ▸") remains in top-right of Business tab canvas.
- Underlying data models and scoring logic are unchanged.
